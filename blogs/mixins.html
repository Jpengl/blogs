    Vue mixin🔥

    Jpeng 2023/9/7 mixin

    # 什么是Mixin

    将组件的公共逻辑或者配置提取出来，哪个组件需要用到时，直接将提取的这部分混入到组件内部即可。这样既可以 减少代码冗余度，也可以让后期维护起来更加容易。

    这里需要注意的是：提取的是逻辑或配置，而不是HTML代码和CSS代码。其实大家也可以换一种想法，mixin就是组件中的组件，Vue组件化让我们的代码复用性更高，那么组件与组件之间还有重复部分，我们使用Mixin在抽离一遍。

    # Mixin和vuex的区别

    Vuex公共状态管理，如果在一个组件中更改了Vuex中的某个数据，那么其它所有引用了Vuex中该数据的组件也会跟着变化。

    Mixin中的数据和方法都是独立的，组件之间使用后是 互相不影响 的。

    # Mixin的定义

    /*
1.根目录创建mixin目录，mixin目录下创建index.js文件

2.定义mixin对象，向外面暴露，这个对象里面可以包含Vue组件中的一些常见配置，如data、methods、created等等。

3.小程序中也可以包含小程序独有的生命周期 onShow() 等等
*/

// 示例： index.js
export const mixins = {
  data() {
    return {
      msg: "我是xxx",
    };
  },
  computed: {},
  created() {
    console.log("我是mixin中的created生命周期函数");
  },
  mounted() {
    console.log("我是mixin中的mounted生命周期函数");
  },
  methods: {
    clickMe() {
      console.log("我是mixin中的点击事件");
    },
  },
};

    # 局部混入

    // 局部混入和组件的按需加载有点类似，就是那里需要用到mixin中的代码时，我们再在相应的组件引入它

// 示例：在App.vue中使用混入  src/App.vue
<template>
  <div id="app">
    <img alt="Vue logo" src="./assets/logo.png" />
    <button @click="clickMe">点击我</button>
  </div>
</template>

<script>
import { mixins } from "./mixin/index"; ** 局部引入混入 **
export default {
  name: "App",
  mixins: [mixins], ** 局部注册使用 **
  components: {},
  created(){
    console.log("组件调用minxi数据",this.msg);
  },
  mounted(){
    console.log("我是组件的mounted生命周期函数")
  }
};
</script>

    # 全局混入

    // 在入口文件引入并注册,后继在其他组件使用时就不需要再次引入和注册，可直接使用
import Vue from "vue";
import App from "./App.vue";
import { mixins } from "./mixin/index";  ** //引入混入 **
Vue.mixin(mixins);  ** //注册混入 **

Vue.config.productionTip = false;

new Vue({
  render: (h) => h(App),
}).$mount("#app");

    请谨慎使用全局混入，因为它会影响每个单独创建的 Vue 实例 (包括第三方组件)。大多数情况下，只应当应用于自定义选项，就像上面示例一样。推荐将其作为插件发布，以避免> 重复应用混入。

    # 选项合并

    mixin中定义的属性或方法的名称与组件中定义的名称冲突时,以组件中的属性和方法为准

    # minxin优缺点

    # 优点

    • 提高代码复用性
    • 无需传递状态
    • 维护方便，只需要修改一个地方即可

    # 缺点

    • 不好追溯源，排查问题稍显麻烦
    • 不能轻易的重复代码
    • 滥用的话后期很难维护
    • 命名冲突
